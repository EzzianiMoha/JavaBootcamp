Comparison Operators

Comparison operators allow us to compare values and make decisions in our programs.

Operator Description Example Result
== Equal to 5 == 5 true
!= Not equal to 5 != 3 true
> Greater than 5 > 3 true
< Less than 5 < 3 false
>= Greater than or equal to 5 >= 5 true
<= Less than or equal to 5 <= 3 false
int x = 10; int y = 5; System.out.println(x == y); // false System.out.println(x != y); // true System.out.println(x > y); // true System.out.println(x < y); // false System.out.println(x >= 10); // true System.out.println(y <= 4); // false

Important: Use == for comparison, not = which is for assignment!

int age = 25; if (age == 18) { // Correct - comparison System.out.println("You are 18!"); } if (age = 18) { // Error - assignment, not comparison System.out.println("This won't work!"); }

If Statements

If statements allow your program to make decisions based on conditions.

Simple If Statement

int age = 20; if (age >= 18) { System.out.println("You are an adult"); }

If-Else Statement

Used when you want to execute one block if true, and another if false.

int score = 75; if (score >= 50) { System.out.println("You passed!"); } else { System.out.println("You failed!"); }

Else-If Statements

Used when you have multiple conditions to check.

int score = 85; if (score >= 90) { System.out.println("Grade: A"); } else if (score >= 80) { System.out.println("Grade: B"); } else if (score >= 70) { System.out.println("Grade: C"); } else if (score >= 60) { System.out.println("Grade: D"); } else { System.out.println("Grade: F"); }

Note: The order of conditions matters! Java evaluates conditions from top to bottom and executes the first block where the condition is true.

Optional Braces for Single Statements

When you have only one statement in an if/else block, you can omit the braces:

int age = 20; // With braces (recommended for beginners) if (age >= 18) { System.out.println("You are an adult"); } // Without braces (works but can be error-prone) if (age >= 18) System.out.println("You are an adult");

Important: When omitting braces, you can only have one statement. Variable declaration and initialization is NOT allowed without braces:

int score = 85; // This works - assigning to existing variable String result; if (score >= 50) result = "Pass"; else result = "Fail"; // This causes ERROR - declaring new variable if (score >= 50) String message = "You passed!"; // COMPILATION ERROR! else String message = "You failed!"; // COMPILATION ERROR!

Nested If Statements

You can put if statements inside other if statements for complex conditions.

int age = 25; boolean hasLicense = true; if (age >= 18) { if (hasLicense) { System.out.println("You can drive!"); } else { System.out.println("You need a license to drive"); } } else { System.out.println("You are too young to drive"); }

Logical Operators

Logical operators allow you to combine multiple conditions.

Operator Description Example Result
&& AND - both conditions must be true (5 > 3) && (2 < 4) true
|| OR - at least one condition must be true (5 > 3) || (2 > 4) true
! NOT - reverses the condition !(5 > 3) false

AND Operator (&&) - Practical Examples

int age = 25; boolean hasLicense = true; if (age >= 18 && hasLicense) { System.out.println("You can drive a car!"); } else { System.out.println("You cannot drive a car"); }

OR Operator (||) - Practical Examples

String day = "Saturday"; if (day.equals("Saturday") || day.equals("Sunday")) { System.out.println("It's the weekend!"); } else { System.out.println("It's a weekday"); }

NOT Operator (!) - Practical Examples

boolean isRaining = false; if (!isRaining) { System.out.println("You don't need an umbrella"); } else { System.out.println("Take an umbrella"); }

Combining Logical Operators

int age = 25; boolean isStudent = true; double income = 20000; if ((age >= 18 && age <= 25) && isStudent && income < 25000) { System.out.println("You qualify for student discount!"); } else { System.out.println("You don't qualify for student discount"); }

Remember: AND (&&) requires ALL conditions to be true, OR (||) requires AT LEAST ONE condition to be true, and NOT (!) reverses the condition.

String Methods

Java provides many useful methods for working with strings. These methods help you manipulate and analyze text data.

Common String Methods

length() - Get String Length

String text = "Hello World"; int length = text.length(); System.out.println("Length: " + length); // Output: Length: 11

charAt() - Get Character at Position

String text = "Java"; char firstChar = text.charAt(0); char lastChar = text.charAt(text.length() - 1); System.out.println("First: " + firstChar); // Output: First: J System.out.println("Last: " + lastChar); // Output: Last: a

indexOf() and lastIndexOf() - Find Character Position

String text = "programming"; int firstM = text.indexOf('m'); int lastM = text.lastIndexOf('m'); System.out.println("First 'm': " + firstM); // Output: First 'm': 6 System.out.println("Last 'm': " + lastM); // Output: Last 'm': 7

toUpperCase() and toLowerCase() - Change Case

String text = "Hello World"; String upper = text.toUpperCase(); String lower = text.toLowerCase(); System.out.println("Upper: " + upper); // Output: Upper: HELLO WORLD System.out.println("Lower: " + lower); // Output: Lower: hello world

trim() - Remove Spaces

String text = " Hello World "; String trimmed = text.trim(); System.out.println("Trimmed: '" + trimmed + "'"); // Output: Trimmed: 'Hello World'

replace() - Replace Characters

String text = "Hello World"; String replaced = text.replace('o', '0'); System.out.println("Replaced: " + replaced); // Output: Replaced: Hell0 W0rld

isEmpty() - Check if String is Empty

String text1 = ""; String text2 = "Hello"; System.out.println("text1 is empty: " + text1.isEmpty()); // Output: true System.out.println("text2 is empty: " + text2.isEmpty()); // Output: false

contains() - Check if String Contains Substring

String text = "Java Programming"; System.out.println("Contains 'Java': " + text.contains("Java")); // Output: true System.out.println("Contains 'Python': " + text.contains("Python")); // Output: false

equals() and equalsIgnoreCase() - Compare Strings

String text1 = "Hello"; String text2 = "hello"; String text3 = "HELLO"; System.out.println("equals: " + text1.equals(text2)); // Output: false (case sensitive) System.out.println("equalsIgnoreCase: " + text1.equalsIgnoreCase(text2)); // Output: true System.out.println("equalsIgnoreCase: " + text1.equalsIgnoreCase(text3)); // Output: true

Important: Always use equals() to compare strings, not ==. The == operator checks if both references point to the same object, while equals() checks if the content is the same.

substring() Method

The substring() method extracts a portion of a string. It's very useful for working with specific parts of text.

substring() Syntax

String substring(int beginIndex) String substring(int beginIndex, int endIndex)

Basic substring() Examples

String text = "Hello World"; // Get substring from index 6 to end String part1 = text.substring(6); System.out.println(part1); // Output: World // Get substring from index 0 to 5 (exclusive) String part2 = text.substring(0, 5); System.out.println(part2); // Output: Hello // Get substring from index 3 to 8 String part3 = text.substring(3, 8); System.out.println(part3); // Output: lo Wo

Important: The endIndex in substring(beginIndex, endIndex) is exclusive - it goes up to but does not include the character at endIndex.

Ternary Operator

The ternary operator is a shorthand way to write simple if-else statements. It makes your code more concise and readable for simple conditions.

Ternary Operator Syntax

condition ? value_if_true : value_if_false

Basic Examples

int age = 20; // Using if-else String message; if (age >= 18) { message = "Adult"; } else { message = "Minor"; } // Using ternary operator String message2 = (age >= 18) ? "Adult" : "Minor"; System.out.println(message); // Output: Adult System.out.println(message2); // Output: Adult

More Practical Examples

int score = 85; // Determine pass/fail String result = (score >= 50) ? "Pass" : "Fail"; System.out.println("Result: " + result); // Output: Result: Pass // Find maximum of two numbers int a = 10, b = 15; int max = (a > b) ? a : b; System.out.println("Maximum: " + max); // Output: Maximum: 15 // Check even/odd int number = 7; String parity = (number % 2 == 0) ? "Even" : "Odd"; System.out.println(number + " is " + parity); // Output: 7 is Odd

Nested Ternary Operator

You can nest ternary operators, but use them sparingly as they can become hard to read.

int score = 85; // Grading system with ternary String grade = (score >= 90) ? "A" : (score >= 80) ? "B" : (score >= 70) ? "C" : (score >= 60) ? "D" : "F"; System.out.println("Grade: " + grade); // Output: Grade: B

Best Practice: Use ternary operators for simple conditions only. For complex logic, stick with if-else statements for better readability.

Switch Statements

Switch statements provide a clean way to handle multiple conditions based on a single value. They're often more readable than long if-else chains when dealing with discrete values.

Switch Statement Syntax

Traditional Switch Syntax

switch (expression) { case value1: // code to execute break; case value2: // code to execute break; default: // code to execute if no cases match }

Enhanced Switch Syntax (Java 14+)

// Statement form (no return value) switch (expression) { case value1 -> statement; case value2, value3 -> statement; default -> statement; } // Expression form (returns a value) dataType result = switch (expression) { case value1 -> expression; case value2, value3 -> expression; default -> expression; }; // With yield for complex logic dataType result = switch (expression) { case value1 -> expression; case value2 -> { // multiple statements yield value; } default -> expression; };

Traditional Switch Statement

The traditional switch statement has been in Java since the beginning and uses case, break, and default keywords.

Basic Switch with Break Statements

int dayNumber = 3; String dayName; switch (dayNumber) { case 1: dayName = "Monday"; break; case 2: dayName = "Tuesday"; break; case 3: dayName = "Wednesday"; break; case 4: dayName = "Thursday"; break; case 5: dayName = "Friday"; break; case 6: dayName = "Saturday"; break; case 7: dayName = "Sunday"; break; default: dayName = "Invalid day"; break; } System.out.println("Day " + dayNumber + " is " + dayName);

Switch with Intentional Fall-Through

Sometimes we intentionally omit break to have multiple cases execute the same code:

String day = "Monday"; String dayType; switch (day) { case "Monday": case "Tuesday": case "Wednesday": case "Thursday": case "Friday": dayType = "Weekday"; break; // This break applies to all weekdays case "Saturday": case "Sunday": dayType = "Weekend"; break; // This break applies to both weekend days default: dayType = "Invalid day"; break; } System.out.println(day + " is a " + dayType);

Important: Forgetting break statements can cause unintended fall-through behavior where multiple cases execute. Always use break unless you intentionally want fall-through.

Enhanced Switch Expression (Java 14+)

Java 14 introduced enhanced switch expressions that are more concise and less error-prone. They use the -> arrow syntax and don't require break statements.

Basic Enhanced Switch

String day = "Saturday"; switch (day) { case "Monday", "Tuesday", "Wednesday", "Thursday", "Friday" -> System.out.println("Weekday"); case "Saturday", "Sunday" -> System.out.println("Weekend"); default -> System.out.println("Invalid day"); }

Returning Values with Enhanced Switch

Enhanced switch can directly return values, making your code more concise:

int month = 3; String season = switch (month) { case 12, 1, 2 -> "Winter"; case 3, 4, 5 -> "Spring"; case 6, 7, 8 -> "Summer"; case 9, 10, 11 -> "Fall"; default -> "Invalid month"; }; System.out.println("Month " + month + " is in " + season);

Using yield for Complex Cases

For more complex logic in enhanced switch, you can use code blocks with yield to return values:

int score = 85; String grade = switch (score / 10) { case 10, 9 -> "A"; case 8 -> "B"; case 7 -> "C"; case 6 -> "D"; default -> { if (score < 0 || score > 100) yield "Invalid score"; else yield "F"; } }; System.out.println("Score " + score + " gets grade: " + grade);

Another yield Example

int number = 15; String description = switch (number) { case 1, 2, 3 -> "Small number"; case 4, 5, 6 -> "Medium number"; case 7, 8, 9 -> "Large number"; default -> { if (number > 100) yield "Very large number"; else if (number < 0) yield "Negative number"; else yield "Other number"; } }; System.out.println(number + " is a " + description);
Feature Traditional Switch Enhanced Switch
Syntax case value: with break case value -> (no break needed)
Multiple values Requires separate case statements Comma-separated: case 1, 2, 3 ->
Return values Cannot return values directly Can return values as expressions
Default case default: default ->
Complex logic Use code blocks with break Use code blocks with yield
Fall-through Happens without break (can be error-prone) No fall-through (safer)

Best Practice: Use enhanced switch expressions (Java 14+) when possible as they are more concise, readable, and less error-prone due to the elimination of fall-through behavior.
