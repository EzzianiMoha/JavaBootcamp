1. Object-Oriented Programming (OOP)

üöÄ What is OOP?

Object-Oriented Programming is a programming paradigm that organizes software design around objects rather than functions and logic.

Four Pillars of OOP

Pillar Description Real-World Example
Encapsulation Bundling data and methods that operate on that data within a single unit (class) A car's dashboard - you can use controls without knowing engine details
Inheritance Creating new classes based on existing classes to reuse code Different types of vehicles (car, truck, motorcycle) inheriting from Vehicle
Polymorphism Objects of different types can be accessed through the same interface Different animals making different sounds when you call "makeSound()"
Abstraction Hiding complex implementation details and showing only essential features Using a phone without knowing how the touchscreen works internally

Understanding Classes and Objects

üìù Class vs Object

Class: A blueprint or template (like a house blueprint)

Object: An instance of a class (like an actual house built from the blueprint)

2. Constructors

üîß What are Constructors?

Constructors are special methods that are automatically called when an object is created. They initialize the object's state.

Default Constructor

public class Car { String brand; String model; int year; // Default constructor - no parameters // If you don't create any constructor, Java provides this automatically public Car() { brand = "Unknown"; model = "Unknown"; year = 2023; System.out.println("Default constructor called!"); } public void displayInfo() { System.out.println(brand + " " + model + " (" + year + ")"); } } public class Main { public static void main(String[] args) { Car car1 = new Car(); // Default constructor is called car1.displayInfo(); // Output: Unknown Unknown (2023) } }

Parameterized Constructor

public class Car { String brand; String model; int year; // Parameterized constructor - takes parameters to initialize object public Car(String carBrand, String carModel, int carYear) { brand = carBrand; model = carModel; year = carYear; System.out.println("Parameterized constructor called!"); } public void displayInfo() { System.out.println(brand + " " + model + " (" + year + ")"); } } public class Main { public static void main(String[] args) { // Using parameterized constructor Car car2 = new Car("Toyota", "Camry", 2022); car2.displayInfo(); // Output: Toyota Camry (2022) } }

3. Constructor Overloading

You can have multiple constructors with different parameters - this is called constructor overloading.

public class Car { String brand; String model; int year; double price; // Constructor 1: Default (no parameters) public Car() { brand = "Unknown"; model = "Unknown"; year = 2023; price = 0.0; System.out.println("Default constructor called"); } // Constructor 2: Brand and model only public Car(String brand, String model) { this.brand = brand; // 'this' refers to current object this.model = model; this.year = 2023; this.price = 0.0; System.out.println("2-parameter constructor called"); } // Constructor 3: All parameters public Car(String brand, String model, int year, double price) { this.brand = brand; this.model = model; this.year = year; this.price = price; System.out.println("Full parameter constructor called"); } public void displayInfo() { System.out.println(brand + " " + model + " (" + year + ") - $" + price); } }

Using different constructors:

public class Main { public static void main(String[] args) { Car car1 = new Car(); // Uses default constructor Car car2 = new Car("Toyota", "Camry"); // Uses 2-parameter constructor Car car3 = new Car("BMW", "X5", 2022, 65000.0); // Uses full constructor car1.displayInfo(); // Unknown Unknown (2023) - $0.0 car2.displayInfo(); // Toyota Camry (2023) - $0.0 car3.displayInfo(); // BMW X5 (2022) - $65000.0 } }

Understanding 'this' keyword: The this keyword refers to the current object instance. It's used to distinguish between instance variables and parameters with the same name. Without 'this', Java would be confused about which variable you mean.

4. Arrays of Objects

üìö Working with Multiple Objects

Just like we can have arrays of numbers or strings, we can have arrays of objects!

Creating and Using Arrays of Objects

public class Student { String name; int grade; public Student(String name, int grade) { this.name = name; this.grade = grade; } public void displayInfo() { System.out.println(name + ": Grade " + grade); } } public class Main { public static void main(String[] args) { // Method 1: Create array then initialize each element Student[] students = new Student[3]; students[0] = new Student("Alice", 85); students[1] = new Student("Bob", 92); students[2] = new Student("Charlie", 78); System.out.println("=== Method 1 ==="); for (int i = 0; i < students.length; i++) { students[i].displayInfo(); } // Method 2: Initialize array with objects directly Student[] students2 = { new Student("Diana", 95), new Student("Eve", 88), new Student("Frank", 91) }; System.out.println("\n=== Method 2 ==="); for (Student student : students2) { student.displayInfo(); } } }

Key Points:

  • Student[] students = new Student[3] creates an array that can hold 3 Student objects
  • Each element must be initialized with new Student() before use
  • Enhanced for loop (for (Student student : students)) is cleaner for iterating

Practical Example: Student Management System

public class StudentManager { public static void main(String[] args) { Student[] students = { new Student("Alice", 85), new Student("Bob", 92), new Student("Charlie", 78), new Student("Diana", 95), new Student("Eve", 88) }; // Calculate average grade double total = 0; for (Student student : students) { total += student.grade; } double average = total / students.length; System.out.println("Average grade: " + average); // Find highest and lowest grades int highest = students[0].grade; int lowest = students[0].grade; String topStudent = students[0].name; String bottomStudent = students[0].name; for (Student student : students) { if (student.grade > highest) { highest = student.grade; topStudent = student.name; } if (student.grade < lowest) { lowest = student.grade; bottomStudent = student.name; } } System.out.println("Top student: " + topStudent + " (" + highest + ")"); System.out.println("Lowest grade: " + bottomStudent + " (" + lowest + ")"); // Count students with grade above 90 int countAbove90 = 0; for (Student student : students) { if (student.grade >= 90) { countAbove90++; } } System.out.println("Students with A grade: " + countAbove90); } }

5. Static Keyword

üè¢ Class-Level vs Object-Level

The static keyword is used for members that belong to the class itself, rather than to individual objects.

üß© Static vs Instance Members

Instance Variables: Each object has its own copy

Static Variables: Shared by all objects of the class

Static Variables (Class Variables)

public class Employee { // Instance variables - each object has its own copy String name; int id; // Static variables - shared by ALL objects static String company = "Tech Corp"; static int employeeCount = 0; public Employee(String name, int id) { this.name = name; this.id = id; employeeCount++; // This increments the SHARED counter } public void displayInfo() { System.out.println("ID: " + id + ", Name: " + name + ", Company: " + company); } // Static method - can be called without creating an object public static void displayEmployeeCount() { System.out.println("Total employees: " + employeeCount); } }

Using Static Members

public class Main { public static void main(String[] args) { // Access static variable WITHOUT creating an object System.out.println("Company: " + Employee.company); // Call static method WITHOUT creating an object Employee.displayEmployeeCount(); System.out.println("\n--- Creating Employees ---"); // Create employees Employee emp1 = new Employee("Alice", 101); Employee emp2 = new Employee("Bob", 102); Employee emp3 = new Employee("Charlie", 103); emp1.displayInfo(); emp2.displayInfo(); emp3.displayInfo(); // Static method shows updated count Employee.displayEmployeeCount(); // Static variable can also be accessed through objects (but not recommended) System.out.println("Company from object: " + emp1.company); } }

Output:

Company: Tech Corp Total employees: 0 --- Creating Employees --- ID: 101, Name: Alice, Company: Tech Corp ID: 102, Name: Bob, Company: Tech Corp ID: 103, Name: Charlie, Company: Tech Corp Total employees: 3 Company from object: Tech Corp

Important Rules for Static Methods:

  • Static methods can only access static variables and call other static methods directly
  • They cannot access instance variables or methods without creating an object first
  • They are called using the class name: ClassName.staticMethod()

Common Use Cases for Static

public class MathUtils { // Constants public static final double PI = 3.14159; // Utility methods public static double calculateCircleArea(double radius) { return PI * radius * radius; } public static int findMax(int[] numbers) { if (numbers.length == 0) return 0; int max = numbers[0]; for (int num : numbers) { if (num > max) max = num; } return max; } // Factory method public static Student createStudent(String name, int grade) { return new Student(name, grade); } } public class Main { public static void main(String[] args) { // Using static constants and methods double area = MathUtils.calculateCircleArea(5.0); System.out.println("Circle area: " + area); int[] nums = {5, 2, 8, 1, 9}; int max = MathUtils.findMax(nums); System.out.println("Maximum: " + max); // Using static factory method Student student = MathUtils.createStudent("John", 85); student.displayInfo(); } }

6. Inheritance

üë®‚Äçüë©‚Äçüëß‚Äçüë¶ Building Class Hierarchies

Inheritance allows a class to inherit properties and methods from another class, promoting code reuse.

Basic Inheritance Example

// Parent class (superclass) - contains common properties and behaviors public class Animal { String name; int age; public Animal(String name, int age) { this.name = name; this.age = age; } public void eat() { System.out.println(name + " is eating."); } public void sleep() { System.out.println(name + " is sleeping."); } public void makeSound() { System.out.println(name + " makes a generic animal sound."); } } // Child class (subclass) - inherits from Animal and adds specific features public class Dog extends Animal { String breed; // Constructor calls parent constructor using super() public Dog(String name, int age, String breed) { super(name, age); // Must be first line in constructor this.breed = breed; } // Additional method specific to Dog public void bark() { System.out.println(name + " is barking! Woof woof!"); } // Method overriding - providing specific implementation @Override public void makeSound() { System.out.println(name + " says: Woof! Woof!"); } } // Another child class public class Cat extends Animal { boolean hasClaws; public Cat(String name, int age, boolean hasClaws) { super(name, age); this.hasClaws = hasClaws; } public void meow() { System.out.println(name + " says meow!"); } @Override public void makeSound() { System.out.println(name + " says: Meow!"); } }

7. Super Keyword - Detailed Explanation

The super keyword is used to refer to the parent class. It has three main uses:

public class Vehicle { String brand; int year; public Vehicle(String brand, int year) { this.brand = brand; this.year = year; } public void start() { System.out.println("Vehicle is starting..."); } public void displayInfo() { System.out.println("Brand: " + brand + ", Year: " + year); } } public class Car extends Vehicle { int doors; String fuelType; public Car(String brand, int year, int doors, String fuelType) { // 1. Calling parent constructor - MUST be first line super(brand, year); this.doors = doors; this.fuelType = fuelType; } @Override public void start() { // 2. Calling parent method super.start(); System.out.println("Car with " + doors + " doors is ready!"); } @Override public void displayInfo() { // 3. Accessing parent's method and adding more info super.displayInfo(); System.out.println("Doors: " + doors + ", Fuel: " + fuelType); } // Additional method specific to Car public void honk() { System.out.println("Beep beep!"); } }

8. Method Overriding - Complete Guide

Method overriding allows a subclass to provide a specific implementation of a method that is already defined in its parent class.

public class Main { public static void main(String[] args) { // Creating objects of different types Animal genericAnimal = new Animal("Generic", 5); Dog dog = new Dog("Buddy", 3, "Golden Retriever"); Cat cat = new Cat("Whiskers", 2, true); System.out.println("=== Testing Inheritance ==="); // All animals can eat and sleep (inherited methods) genericAnimal.eat(); dog.eat(); // Inherited from Animal cat.eat(); // Inherited from Animal System.out.println("\n=== Testing Method Overriding ==="); // Each animal makes a different sound (overridden methods) genericAnimal.makeSound(); // Generic animal sound dog.makeSound(); // Woof! Woof! (overridden) cat.makeSound(); // Meow! (overridden) System.out.println("\n=== Testing Specific Methods ==="); // Specific methods only available in subclasses dog.bark(); // Only Dog can bark cat.meow(); // Only Cat can meow // genericAnimal.bark(); // This would cause ERROR! System.out.println("\n=== Testing Car Class ==="); Car car = new Car("Toyota", 2022, 4, "Gasoline"); car.displayInfo(); car.start(); car.honk(); } }

Rules for Method Overriding:

  1. The method must have the same name and parameters as in the parent class
  2. The return type should be the same or a subtype (covariant return type)
  3. Cannot reduce visibility (e.g., can't change from public to private)
  4. Use @Override annotation for clarity and compile-time checking

Interfaces and Polymorphism

Interfaces: Defining Behavioral Contracts

An interface defines a contract that classes must fulfill by implementing specified methods.

Interface Declaration and Implementation

// Interface for user authentication behaviors interface Authenticatable { boolean login(String username, String password); void logout(); boolean isLoggedIn(); } // Student implementation class Student implements Authenticatable { private String studentId; private boolean loggedIn = false; public Student(String studentId) { this.studentId = studentId; } @Override public boolean login(String username, String password) { if (username.equals("student") && password.equals("pass123")) { loggedIn = true; System.out.println("Student " + studentId + " logged in"); return true; } return false; } @Override public void logout() { loggedIn = false; System.out.println("Student logged out"); } @Override public boolean isLoggedIn() { return loggedIn; } } // Teacher implementation class Teacher implements Authenticatable { private String teacherId; private boolean loggedIn = false; public Teacher(String teacherId) { this.teacherId = teacherId; } @Override public boolean login(String username, String password) { if (username.equals("teacher") && password.equals("teach456")) { loggedIn = true; System.out.println("Teacher " + teacherId + " logged in"); return true; } return false; } @Override public void logout() { loggedIn = false; System.out.println("Teacher logged out"); } @Override public boolean isLoggedIn() { return loggedIn; } }

Interface Characteristics:

  • Defines method signatures without implementations
  • Classes implement interfaces using the implements keyword
  • Multiple interfaces can be implemented by a single class
  • Promotes code abstraction and standardization

Polymorphism: Interface-Based Type System

public class SchoolSystem { public static void main(String[] args) { // Polymorphic array: Authenticatable reference type, various implementations Authenticatable[] users = { new Student("S1001"), new Teacher("T2001"), new Student("S1002") }; // Polymorphic method invocation System.out.println("=== User Authentication ==="); for (Authenticatable user : users) { boolean success = user.login("student", "pass123"); System.out.println("Login successful: " + success); System.out.println("User logged in: " + user.isLoggedIn()); System.out.println(); } // Demonstrating interface compatibility System.out.println("=== Individual User Processing ==="); processUser(new Student("S1003")); processUser(new Teacher("T2002")); } // Method accepting any Authenticatable implementation public static void processUser(Authenticatable user) { System.out.println("Processing user..."); user.login("user", "password"); System.out.println("Currently logged in: " + user.isLoggedIn()); } }

Advanced Polymorphism with Multiple Interfaces

// Additional interface for grading capability interface Grader { void assignGrade(String studentId, int grade); double calculateAverage(); } // Admin class implementing multiple interfaces class Admin implements Authenticatable, Grader { private String adminId; private boolean loggedIn = false; public Admin(String adminId) { this.adminId = adminId; } @Override public boolean login(String username, String password) { if (username.equals("admin") && password.equals("admin789")) { loggedIn = true; System.out.println("Admin " + adminId + " logged in"); return true; } return false; } @Override public void logout() { loggedIn = false; System.out.println("Admin logged out"); } @Override public boolean isLoggedIn() { return loggedIn; } @Override public void assignGrade(String studentId, int grade) { System.out.println("Admin assigned grade " + grade + " to student " + studentId); } @Override public double calculateAverage() { return 85.5; // Example calculation } } // Demonstrating interface polymorphism public class UserManagement { public static void main(String[] args) { Admin admin = new Admin("A3001"); // Interface reference variables Authenticatable authUser = admin; Grader grader = admin; // Access through different interfaces System.out.println("=== Authentication Interface ==="); authUser.login("admin", "admin789"); System.out.println("Is logged in: " + authUser.isLoggedIn()); System.out.println("\n=== Grader Interface ==="); grader.assignGrade("S1005", 95); System.out.println("Class average: " + grader.calculateAverage()); // Demonstrating type checking System.out.println("\n=== Type Compatibility ==="); if (authUser instanceof Grader) { System.out.println("This user can also assign grades"); Grader g = (Grader) authUser; g.assignGrade("S1006", 88); } } }

Polymorphism Principles:

  1. Interface Polymorphism: Objects can be accessed through interface references
  2. Runtime Binding: Method implementation determined at execution time
  3. Type Substitution: Interface references can hold any implementing object
  4. Extensibility: New implementations can be added without modifying existing code

Key Benefits of Interface-Based Design

  • Decoupling: Implementation separated from interface
  • Testability: Easy mocking and testing through interfaces
  • Maintainability: Changes to implementation don't affect interface consumers
  • Flexibility: Multiple implementations can coexist
  • Type Safety: Compile-time checking of interface compliance

10. Getters & Setters - Made Simple!

üîí Why Use Getters and Setters?

They protect your data and give you control over how it's accessed and modified.

Simple Student Class Example

public class SimpleStudent { // Private variables - cannot be accessed directly private String name; private int age; private double gpa; // Constructor public SimpleStudent(String name, int age, double gpa) { this.name = name; setAge(age); // Use setter to validate setGpa(gpa); // Use setter to validate } // üì• GETTERS - Allow READ access public String getName() { return name; } public int getAge() { return age; } public double getGpa() { return gpa; } // üì§ SETTERS - Allow WRITE access with validation public void setName(String name) { if (name != null && !name.trim().isEmpty()) { this.name = name; } else { System.out.println("Error: Name cannot be empty!"); } } public void setAge(int age) { if (age >= 0 && age <= 120) { // Reasonable age validation this.age = age; } else { System.out.println("Error: Age must be between 0 and 120!"); } } public void setGpa(double gpa) { if (gpa >= 0.0 && gpa <= 4.0) { // GPA must be 0.0 to 4.0 this.gpa = gpa; } else { System.out.println("Error: GPA must be between 0.0 and 4.0!"); } } // Helper method using getters public void displayInfo() { System.out.println("Student: " + getName() + ", Age: " + getAge() + ", GPA: " + getGpa()); } }

public class Rectangle { private double length; private double width; public Rectangle(double length, double width) { setLength(length); setWidth(width); } // Getter for length public double getLength() { return length; } // Setter for length public void setLength(double length) { if (length > 0) { this.length = length; } } // Getter for width public double getWidth() { return width; } // Setter for width public void setWidth(double width) { if (width > 0) { this.width = width; } } // Calculate area using getters public double getArea() { return getLength() * getWidth(); // Using getters } // Calculate perimeter public double getPerimeter() { return 2 * (length + width); // Using direct access (inside class) } public void display() { System.out.println("Rectangle: " + length + " x " + width); System.out.println("Area: " + getArea()); System.out.println("Perimeter: " + getPerimeter()); } }

Quick Summary:

  • ‚úÖ Getter: Returns the value ‚Üí public Type getVariable()

  • ‚úÖ Why use? To add validation and control
  • ‚úÖ Tip: Don't always make setters - some data shouldn't be changed!
