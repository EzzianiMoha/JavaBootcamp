üìù Detailed Requirements Step-by-Step

1. Abstract Person Class

  • Create abstract class Person
  • Instance variables: id, name, age, phone (all private)
  • Static variable: nextId (for auto-generating IDs)
  • Constructor: Takes name, age, phone; auto-generates ID
  • Abstract method: getContact() - returns contact information
  • Concrete method: displayInfo() - shows basic information
  • Getters and setters for all private variables

2. Patient Class (extends Person)

  • Additional variables: patientId, bloodGroup, allergies (array), isAdmitted
  • Constructor: Calls super() and initializes Patient-specific variables
  • Override getContact(): Returns "Patient Contact: " + phone
  • Methods:
    • scheduleAppointment(doctor, date, time)
    • displayMedicalInfo() - shows blood group and allergies
    • addAllergy(allergy) - adds to allergies array
    • admitToHospital() / dischargeFromHospital()

3. Doctor Class (extends Person)

  • Additional variables: doctorId, specialization, consultationFee, yearsExperience
  • Static variable: totalDoctors (counts all doctors)
  • Constructor: Increments totalDoctors, auto-generates doctorId like "DOC001"
  • Override getContact(): Returns "Dr. " + name + " - " + phone
  • Methods:
    • prescribeMedicine(patient, medicine)
    • displaySchedule() - shows doctor's availability
    • calculateFee(hours) - returns consultationFee * hours
  • Static method: displayTotalDoctors()

4. Appointment Class

  • Instance variables: appointmentId, patient (Patient object), doctor (Doctor object), date, time, status
  • Static variable: nextAppointmentId
  • Constructor: Takes patient, doctor, date, time; auto-generates ID
  • Methods:
    • displayAppointment() - shows all appointment details
    • cancelAppointment() - changes status to "Cancelled"
    • completeAppointment() - changes status to "Completed"
  • Getters for all variables

5. Main Program (HospitalManagementSystem.java)

  • Create arrays of Patient, Doctor, and Appointment objects
  • Demonstrate polymorphism with Person[] array
  • Test all methods from all classes
  • Show static variable usage
  • Display hospital statistics

üìö Concepts to Review Before Starting

  • Abstract Classes: Classes that cannot be instantiated, can have abstract methods
  • Inheritance: Child classes inherit from parent using extends
  • Polymorphism: Parent reference can hold child objects
  • Static: Variables/methods that belong to the class, not objects
  • Arrays of Objects: Remember to initialize with new ClassName[size]
  • Constructors: Use super() to call parent constructor
